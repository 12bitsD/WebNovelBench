# 网文质量评估 Benchmark v2：基于叙事因果链的深度设计

> v1 是"评什么"，v2 是"为什么这样评"和"怎么精确评"。
> 核心变化：从"列举维度"转向"理解机制→定义标准→设计度量"。

---

## 零、核心问题重构

之前的思路：好的作品有哪些特征？→ 列出维度 → 打分

正确的思路：
  好的作品做了什么（具体技法）→
  这些技法产生了什么效果（因果链）→
  这些效果如何被度量（评估标准）→
  度量的精度和可靠性如何保证（方法论）

一个故事本质上是一台**响应生成机器**：
  输入：文字序列
  过程：通过特定技法操控读者的认知和情感
  输出：特定的阅读体验（好奇、紧张、感动、震撼、思考）

所以评估的本质是：**这台机器的工艺水平如何？**

## 一、叙事因果链：从技法到效果

### 1.1 信息流管理

**这是最核心的叙事机制。所有好的故事，本质上都在管理"读者知道什么"和"角色知道什么"之间的差距。**

因果链：

  信息缺口（gap）→ 好奇心（curiosity）→ 阅读动力
       ↓
  信息揭示（reveal）→ 惊讶（surprise）→ 情感冲击
       ↓
  信息重组（recontextualization）→ 重新理解（insight）→ 智识满足

**这不是一个维度，而是一个系统。** 评估的不是"有没有伏笔"，而是这个信息流系统的精密程度。

#### 从各作品看信息管理的三个层次：

**层次一：单点伏笔（最基础）**
  定义：一个细节在后文被回收
  例：进击的巨人第1话墙上的文字 → 第90话揭示含义
  评估标准：回收率 = 被回收的伏笔数 / 有意识埋设的伏笔数

**层次二：信息级联（中等）**
  定义：一个揭示同时解答旧问题+提出新问题
  例：进击的巨人揭示"墙外有人类" → 解答"墙从哪来" → 提出"墙外人类是谁"
  评估标准：级联深度 = 一个揭示触发的后续问题链长度

**层次三：叙事重构（最高）**
  定义：揭示改变了读者对整个故事的理解框架
  例：进击的巨人"马莱篇"——读者发现自己一直在"敌方视角"看故事
  例：道诡异仙——读者发现"现实"可能只是另一层幻觉
  评估标准：框架转变次数 + 每次转变后读者需要重新理解的范围

#### 度量方法：

```
度量1：伏笔-回收矩阵
  提取文本中所有"看似不寻常"的细节（用 LLM）
  追踪每个细节是否在后文被明确回应
  计算：
    早期伏笔密度 = 前20%章节的伏笔数 / 总字数
    回收率 = 被回收的伏笔数 / 总伏笔数
    回收延迟 = 平均相隔多少字后回收
    意外度 = 回收方式是否在读者预期之外（LLM Judge）

度量2：信息张力曲线
  沿文本滑动窗口，每个窗口计算：
    窗口内提出但未解答的问题数（Q_pending）
    窗口内解答的问题数（Q_resolved）
    窗口内新提出的问题数（Q_new）
  张力 = Q_pending / (Q_resolved + 1)
  好的故事：张力曲线有规律波动，整体不趋近于0也不爆炸
  烂的故事：张力一直很低（没悬念）或一直很高（坑太多填不了）

度量3：框架转变检测
  用 LLM 对每10%的文本做"世界观摘要"
  比较相邻摘要的差异度
  差异度超过阈值 = 发生了框架转变
```

#### 评分标准：

```
1-2分：无伏笔，信息线性释放，无悬念管理
3-4分：有零散伏笔，但回收率低（<50%），无级联
5-6分：伏笔系统化，回收率高（>70%），有简单级联
7-8分：信息级联精密，多个揭示相互嵌套，读者不断被重新定位
9-10分：叙事重构级别——揭示改变了整个故事的理解框架，
        且重构是"意料之外，情理之中"的
```

### 1.2 角色弧光系统

**角色不是"有成长就好"。评估的不是"变没变"，而是"变化是否被故事逻辑驱动，是否与主题共鸣"。**

因果链：

  初始状态（establishing）→ 读者形成预期
       ↓
  压力/冲突（complication）→ 预期受到挑战
       ↓
  关键选择（crisis）→ 角色在压力下做出决定
       ↓
  转变（transformation）→ 决定导致不可逆的变化
       ↓
  新状态（resolution）→ 读者理解变化的意义

**核心判断：转变是"故事驱动"还是"作者强行"？**

#### 从各作品看角色弧光的类型：

**正向弧光（成长型）**
  例：艾伦·耶格尔（进击的巨人）——从"要杀光所有巨人"到理解敌人也是人
  关键：每一步变化都有具体事件驱动，不是突然"顿悟"

**负向弧光（堕落型）**
  例：夜神月（死亡笔记）——从"要用笔记改变世界"到"我是新世界的神"
  关键：堕落是渐进的，每一步都"合理"

**扁平弧光（信念型）**
  例：爱德华·艾尔利克（钢炼）——信念不变，但理解不断深化
  关键：不是"没变化"，而是"在变化的环境中坚守，在坚守中成长"

**颠覆弧光（认知型）**
  例：克莱恩·莫雷蒂（诡秘之主）——从"想回家"到"承担世界的责任"
  关键：变化不是能力提升，而是身份认同的重塑

#### 度量方法：

```
度量1：角色属性追踪
  对每个主要角色，提取以下属性在不同阶段的值：
    - 核心信念（belief）
    - 关系定位（relationship_focus）
    - 决策模式（decision_pattern）
    - 情感基调（emotional_tone）
  用 LLM 在每10%的文本处提取，计算属性向量的变化幅度

度量2：选择-后果链追踪
  提取角色做出的每个关键选择
  每个选择追溯：
    a) 选择的原因是什么？（是否有前文铺垫）
    b) 选择的后果是什么？（是否在后文体现）
    c) 后果是否影响了后续选择？（是否有因果链）
  评分 = 有完整因果链的选择数 / 总关键选择数

度量3：弧光-主题耦合度
  角色的变化是否与故事主题直接相关？
  例：钢炼——爱德华理解"等价交换不是万能的"与主题"有些东西无法交换"完美耦合
  例：进击的巨人——艾伦的堕落与主题"自由的代价"完美耦合
  用 LLM Judge 评估耦合程度（1-10）

度量4：非主角弧光检测
  不是只看主角——配角是否有自己的弧光？
  例：钢炼中马斯·休斯的死亡改变了整个故事的基调
  例：进击的巨人中莱纳的双重人格
  评分 = 有独立弧光的配角数 / 主要配角总数
```

#### 评分标准：

```
1-2分：角色是工具人，无变化，无动机，纯功能
3-4分：有变化但突兀（"突然就领悟了"），无铺垫
5-6分：有合理弧光，变化有事件驱动，但深度一般
7-8分：弧光完整，与主题深度耦合，配角也有弧光
9-10分：弧光是故事的核心驱动力，变化是不可逆的、深刻的、
        读者能在角色身上看到自己的
```

### 1.3 结构张力引擎

**故事的"好看"本质上是张力的管理——什么时候紧张，什么时候释放，什么时候反转。**

因果链：

  稳态（equilibrium）→ 读者舒适
       ↓
  扰动（disruption）→ 舒适被打破 → 张力上升
       ↓
  升温（escalation）→ 问题越来越严重 → 张力持续上升
       ↓
  危机（crisis）→ 到达临界点 → 张力最高
       ↓
  转折（turning point）→ 方向改变 → 张力释放或重构
       ↓
  新稳态 → 但比之前更高一层 → 为下一轮张力蓄力

**这不是"三幕式"——这是所有叙事的底层引擎。**

#### 从各作品看张力管理的差异：

**JOJO 的张力：局部精密**
  每场战斗都是一个微型张力引擎：
    遇到新替身 → 不知道能力 → 被压制 → 发现弱点 → 反转
  特点：张力来自"创意限制"而非"力量差距"
  评估：每场冲突是否遵循"问题→探索→解决"的节奏

**进击的巨人 的张力：全局递进**
  每一季都是上一季的升级版：
    墙内生存 → 墙外真相 → 马莱视角 → 地鸣
  特点：张力不是重复，而是层层递进
  评估：后期冲突是否比前期更有深度（不只是更大）

**诡秘之主 的张力：信息差驱动**
  主角知道的比读者多（或少），信息差本身制造张力
  特点：张力来自"我知道你不知道"而非"打不打得过"
  评估：信息差是否被有效利用

#### 度量方法：

```
度量1：情感波动曲线
  用 emotion detection 模型沿文本滑动
  提取：兴奋度、紧张度、悲伤度、恐惧度
  计算：
    波动幅度 = 标准差 / 均值（变异系数）
    波动频率 = 方向变化次数 / 总长度
    波动层次 = 是否有多个时间尺度的波动叠加
      （章节级小波动 + 卷级中波动 + 全局大波动）

度量2：冲突升级检测
  提取每个冲突/战斗/对抗
  分析：
    冲突的赌注是否在升级？（从个人→团队→世界）
    冲突的复杂度是否在升级？（从简单对抗→多方博弈）
    冲突的代价是否在增加？（有没有真正的损失）
  评分 = 升级的维度数 / 可升级的维度总数

度量3：节奏密度分析
  将文本按段落分类：action / dialogue / description / inner_monologue
  分析：
    各类型的分布是否合理（不是全是战斗或全是旁白）
    是否有节奏变化（高潮段密集action，缓冲段增加dialogue）
    "灌水指数" = 信息密度低于阈值的连续段落数 / 总段落数

度量4：钩子有效性
  提取每个章节结尾的最后一段
  用 LLM Judge 评估：
    这个结尾是否让人想继续读？（1-10）
    类型：悬念 / 反转 / 新信息 / 情感冲击 / 无钩子
  计算：有效钩子比例（分数≥6的比例）
```

#### 评分标准：

```
1-2分：节奏平坦，无波动，无冲突升级
3-4分：有基本节奏但单调（一直在高潮或一直平淡）
5-6分：有合理节奏，但层次单一（只有章节级波动）
7-8分：多层次节奏叠加，冲突持续升级，有真正的损失
9-10分：节奏如同交响乐——多个声部交织，张力曲线本身成为叙事的一部分
```

### 1.4 世界观构建

**世界观不是"设定集"——它是一个"可能性空间"，决定了故事中什么事情可以发生。**

因果链：

  基础规则（axiom）→ 世界的"物理定律"
       ↓
  规则展开（implication）→ 规则的具体后果
       ↓
  规则交互（interaction）→ 多条规则碰撞产生意外结果
       ↓
  规则突破（transcendence）→ 在极端情况下规则被挑战/重定义

**好的世界观：从少量基础规则推导出丰富的可能性**
**烂的世界观：大量零散设定，互相不关联，随时可以打破**

#### 从各作品看世界观构建的差异：

**钢炼 的世界观：极致简洁**
  一条规则："等价交换"——物质不灭，等价转换
  从这一条推导出：炼金术的全部体系、故事的哲学主题、角色的行为逻辑
  评估：设定的"推导比" = 能从基础规则推导出的设定数 / 总设定数

**JOJO 的世界观：约束创造精彩**
  替身能力不是"强弱"，而是"功能"——每个替身有独特能力
  好的替身：能力受限但创意十足（如：杀手皇后——只能触碰引爆）
  烂的替身：能力无限（如：很多后期替身越来越抽象）
  评估：能力系统的"约束密度" = 能力的限制条件数 / 能力的功能数

**诡秘之主 的世界观：系统性最强**
  22条途径 × 10个序列 = 220种能力位置
  每个位置有明确的能力、限制、代价
  评估：设定的覆盖率 = 有明确定义的能力数 / 总能力数

**斗破苍穹 的反面教材：力量体系崩坏**
  斗者→斗师→...→斗帝，等级越多越空洞
  后期：斗帝之上还有什么？新等级不断被加出来
  评估：是否有"天花板"，规则是否始终被遵守

#### 度量方法：

```
度量1：设定一致性检测
  提取世界观的核心规则（用 LLM）
  搜索文本中是否存在违反规则的描述
  一致性 = 1 - (违反次数 / 规则引用总次数)

度量2：推导密度
  提取基础规则（如"等价交换"）
  检测故事中有多少设定是从基础规则推导出来的
  推导密度 = 推导出的设定数 / 基础规则数
  （钢炼：一条规则推导出几乎全部设定 → 极高）

度量3：天花板检测
  检测力量体系是否有上限
  方法：提取力量等级描述，检查是否存在"最高级之上还有更高级"
  有天花板 = +分，无限膨胀 = -分

度量4：规则展示方式
  世界观是通过"旁白解释"展示还是"角色行动"展示？
  例：钢炼通过爱德华的炼金术行动展示规则，而非大段旁白
  度量：旁白解释的比例 vs 行动展示的比例
```

### 1.5 主题共振系统

**主题不是"故事告诉读者什么道理"——而是"故事的结构本身在说什么"。**

因果链：

  控制思想（controlling idea）→ 故事的核心论点
       ↓
  二元对立（opposition）→ 控制思想的对立面
       ↓
  辩证展开（dialectic）→ 两个对立面在故事中碰撞
       ↓
  综合（synthesis）→ 故事结尾的立场（但好的故事不简单站队）

**关键区分：**
  烂的主题表达：角色说教（"我们要珍惜和平"）
  好的主题表达：结构即主题（钢炼的"等价交换"——你真的不能什么都要）

#### 从各作品看主题深度的差异：

**浅层主题（善恶对立）**
  例：大部分网文——好人打坏人，正义必胜
  特点：主题不需要思考，读者天然认同

**中层主题（道德灰度）**
  例：进击的巨人——"自由"到底是什么？为了自由可以杀多少人？
  特点：没有简单答案，读者需要自己思考

**深层主题（存在主义）**
  例：道诡异仙——什么是真实？什么是自我？认知的本质是什么？
  特点：触及人类存在的根本问题

**主题-结构统一（最高境界）**
  例：钢炼——主题"等价交换"就是力量体系，力量体系就是主题
  例：进击的巨人——主题"自由的代价"就是整个故事的走向
  特点：删掉任何一部分，主题都不完整

#### 度量方法：

```
度量1：主题复杂度分析
  用 LLM 提取故事的核心主题
  分类：
    - 单一主题 vs 多主题交织
    - 是否有对立主题（如：自由 vs 安全）
    - 主题的抽象层次（具体→一般→哲学）
  复杂度 = 对立主题数 × 抽象层次

度量2：说教检测（负面指标）
  搜索直接表达主题的"说教段落"
  如："原来，真正的XX是YY啊" 这类内心独白
  说教密度 = 说教段落数 / 总段落数
  高说教 = 低分（好的故事通过结构展示主题，不通过角色嘴巴）

度量3：主题-结构耦合度
  问 LLM：
    a) 如果删掉主角的成长线，主题是否还完整？
    b) 如果删掉反派的动机，主题是否还完整？
    c) 如果改变结局，主题是否改变？
  耦合度 = "改变会影响主题"的回答比例

度量4：主题持久性
  主题是否贯穿始终，还是只在结尾才出现？
  方法：每10%的文本提取主题，计算主题的一致性
  好的故事：主题从第10%就存在，持续到结尾
  烂的故事：主题只在最后几章突然出现（"原来这个故事是关于爱啊"）
```

## 二、方法论→维度→度量的完整映射

| 创作方法论 | 产生效果 | 评估维度 | 核心度量 | 评分关键 |
|-----------|---------|---------|---------|---------|
| 谜题盒(进击巨人) | 持续好奇→智识满足 | 信息流管理 | 伏笔回收率+级联深度+框架转变 | 意料之外+情理之中 |
| 等价交换(钢炼) | 结构统一感→主题共鸣 | 主题-结构耦合 | 说教密度+耦合度+持久性 | 不说教，结构即主题 |
| 黄金螺旋(JOJO) | 创意满足→策略快感 | 冲突设计 | 约束密度+策略层级+能力创意度 | 限制创造精彩 |
| 蓝图法(诡秘之主) | 世界可信→沉浸感 | 世界观构建 | 一致性+推导密度+天花板 | 系统自洽 |
| 叙事欺骗(道诡异仙) | 认知颠覆→深度思考 | 叙事层次 | 框架转变次数+不确定性管理 | 颠覆但合理 |
| 日更连载(网文本质) | 追更动力→持续阅读 | 节奏+钩子 | 波动曲线+钩子有效性+灌水指数 | 张弛有度 |

## 三、评估的精度问题

### 3.1 为什么 LLM Judge 不够

LLM Judge 的根本问题：它评估的是"文本描述的质量"，而不是"阅读体验的质量"。

  一个文本可能结构完美但读起来无聊
  一个文本可能有缺陷但读起来很爽

所以需要**双轨评估**：
  轨道A：结构分析（LLM Judge 擅长）
  轨道B：体验模拟（需要人类校准）

### 3.2 体验模拟的可能方法

```
方法1：阅读时长预测
  预测一个普通读者读完需要多长时间
  如果预测时长 >> 实际时长 → 说明内容密度低（灌水）
  如果预测时长 << 实际时长 → 说明内容密度高

方法2：续读意愿测试
  在多个截断点问 LLM："你想继续读吗？为什么？"
  续读意愿曲线 = 读者粘性的代理指标

方法3：情感响应模拟
  用情感模型模拟读者在每个段落的情感状态
  与作者意图的情感曲线对比
  重合度 = 情感设计的精度

方法4：记忆留存预测
  问 LLM："读完后你会记住什么？"
  预测的记忆点 vs 作者设计的记忆点
  重合度 = 叙事设计的有效性
```

### 3.3 人类校准的必要性

自动评分最终必须通过人类校准验证：

```
校准流程：
1. 选取 20 部作品（S/A/B 各档都有）
2. 每部作品找 3-5 位资深读者评分
3. 人类评分 vs 自动评分 的相关性
4. 如果 Spearman ρ > 0.7 → 自动评分可信
5. 分析差异最大的案例 → 修正 Judge prompt
```

## 四、网文特有维度的深度分析

### 4.1 爽点机制

**爽点不是"主角赢了"——爽点是"读者预期被满足或被超越的瞬间"。**

因果链：
  预期建立（读者知道主角会赢）→ 期待
       ↓
  过程展示（怎么赢）→ 悬念
       ↓
  满足/超越（赢了，但方式出乎意料）→ 爽感

**好的爽点：过程有创意，满足有层次**
  例：JOJO——DIO战，乔瑟夫的"你下一步会..."
  例：诡秘之主——克莱恩用信息差碾压，读者和敌人同时被震惊

**烂的爽点：纯力量碾压，无过程**
  例：大量系统文——"叮！获得XX神器，秒杀全场"

#### 度量方法：

```
爽点质量 = 过程创意度 × 满足层次

过程创意度（LLM Judge, 1-10）：
  - 纯力量碾压：1-2
  - 有简单策略：3-5
  - 有创意策略/反转：6-8
  - 策略与角色特质深度绑定：9-10

满足层次（分类）：
  - Level 1: 主角变强了（低满足）
  - Level 2: 主角赢了（中满足）
  - Level 3: 赢的方式揭示了新信息（高满足）
  - Level 4: 赢改变了故事走向（极高满足）
```

### 4.2 灌水检测

**灌水 = 单位字数内的有效信息密度低于阈值。**

但"有效信息"需要定义：
  - 情节推进信息：改变了故事状态的信息
  - 角色发展信息：改变了角色认知/关系的信息
  - 世界观信息：扩展了读者对世界的理解
  - 情感信息：触发了情感反应的信息

**不算"有效信息"的：**
  - 重复描述（"他使出了XX拳，威力惊人" × 5次）
  - 无意义对话（不影响情节也不发展角色的对话）
  - 旁白注释（角色内心重复已经展示过的内容）

#### 度量方法：

```
信息密度 = (情节推进段落数 + 角色发展段落数 + 世界观扩展段落数) / 总段落数

检测方法：
1. 用 LLM 将每个段落分类为：
   - P（情节推进）：改变了故事状态
   - C（角色发展）：改变了角色认知/关系
   - W（世界观）：扩展了世界理解
   - E（情感）：触发了情感反应
   - N（填充）：以上都不是

2. 灌水指数 = N段落数 / 总段落数

3. 灌水集中度：是否集中在某些章节（中后期灌水是网文通病）

4. 灌水趋势：灌水指数是否随章节递增（越写越水）
```

### 4.3 连载稳定性

**连载作品的质量波动是一个独立的评估维度。**

```
稳定性指标：
  1. 分数方差：按章节/卷分段评分，计算方差
  2. 趋势斜率：后半部分分数是否显著低于前半部分
  3. 烂尾指数：最后20%的平均分数 vs 前80%的平均分数

好的连载：
  诡秘之主：分数方差小，趋势平坦
  进击的巨人：分数有波动但整体上升（越来越精彩）

烂的连载：
  斗破苍穹：后半部分分数显著下降
  大量网文：前100章精彩，后面断崖式下跌
```

## 五、评测集设计的深度思考

### 5.1 选择标准：不只是"好"和"烂"

要选择能**拉开区分度**的作品：

```
S级（必须有）：
  - 结构精密型：诡秘之主、钢炼、进击的巨人
  - 创意突破型：道诡异仙、JOJO
  - 系统完美型：凡人修仙传（前中期）
  - 情感共鸣型：CLANNAD、未闻花名

A级（必须有）：
  - 有明显优点但有缺陷的：
    - 斗破苍穹（开头极好，后期崩坏）
    - 完美世界（世界观宏大但力量膨胀）
    - 一拳超人（创意极好但后期重复）

B级（必须有）：
  - 典型问题作品：
    - 纯模板化系统文
    - 灌水严重的长篇玄幻
    - 角色工具人的后宫文

关键：B级作品的问题必须是**可诊断的**——
  不是"我不喜欢"，而是"它在具体维度上有可测量的缺陷"
```

### 5.2 采样策略的深度思考

不同维度需要不同的采样策略：

```
信息流管理：需要全书采样（伏笔可能跨越100万字）
  → 每5%采样一次，每次5000字

角色弧光：需要关键节点采样
  → 开头 + 第一次重大选择 + 中期转变 + 高潮 + 结尾

节奏分析：需要连续采样
  → 选取5个连续章节（1-2万字）做精细分析

主题分析：需要分散采样
  → 每20%采样一次，每次3000字

灌水检测：需要全书均匀采样
  → 每章取最后一段（灌水通常在章末）
```

## 六、评估的终极问题

### 这个 benchmark 到底在评什么？

不是在评"好不好看"——那太主观。

而是在评：**这台叙事机器的工艺水平。**

具体来说：
  1. 设计精度：作者对信息释放的控制有多精确？
  2. 执行一致性：100万字内是否保持了设计意图？
  3. 创新程度：在类型框架内提供了多少新东西？
  4. 深度-可达性：主题有多深，同时有多容易被读者感知？

这四个指标加在一起，就是"叙事工艺水平"。

它不等于"读者喜欢程度"（那受市场营销、类型偏好、个人口味影响），
但它与"读者喜欢程度"有很强的相关性——
因为好的工艺会让更多的读者在更多的维度上获得更深的体验。

## 七、实施的优先级

```
P0（必须先做）：
  - 信息流管理的度量方法（最核心的维度）
  - 角色弧光的追踪方法（最能区分好坏的维度）
  - LLM Judge prompt 的校准（决定评估质量）

P1（紧随其后）：
  - 节奏分析的自动检测
  - 世界观一致性的自动检测
  - 评测集的构建（选作品 + 采样）

P2（迭代完善）：
  - 人类校准
  - 评分权重优化
  - 网文特有维度（爽点、灌水、连载稳定性）

P3（长期优化）：
  - 跨语言评测（中文→日文→英文）
  - 类型细分（不同类型的权重调整）
  - 动态更新（新作品加入评测集）
```
