# 网文质量评估 Benchmark 设计方案

> 基于叙事创作方法论的深度重构
> 目标：验证一部作品是否是"好作品"

---

## 一、从创作方法论反推评估维度

要评估"好作品"，必须先理解"好作品是怎么被创造出来的"。
以下是各顶级作品的核心创作方法论，以及它们揭示的评估维度。

### 1.1 进击的巨人 — "谜题盒"叙事法

**方法论：**
- 谏山创从第一话就规划了完整的世界观（墙→人类→巨人→坐标→尤弥尔→自由）
- 每一次"真相揭露"都同时解答旧问题+提出新问题
- 不可靠叙事者：读者看到的"真相"不断被颠覆
- 主题先行：自由 vs 安乐死、战争循环、偏见与仇恨

**揭示的评估维度：**
- [D1] 信息管理：作者向读者释放信息的节奏
- [D2] 伏笔密度与回收率：埋了多少坑，填了多少
- [D3] 颠覆质量：反转是否"意料之外，情理之中"
- [D4] 主题一致性：所有情节是否服务于核心主题

### 1.2 钢之炼金术师 — "等价交换"结构法

**方法论：**
- 荒川弘从一开始就知道结局（人造人→父亲大人→烧瓶中的小人）
- "等价交换"既是力量体系也是哲学主题——结构与内容完美统一
- 每个角色都有完整弧光（尤其马斯·休斯、大哥哥、人造人们）
- 支线从不偏离主线：每个支线都在为主线蓄力
- 幽默与严肃的精确切换

**揭示的评估维度：**
- [D5] 力量体系与主题的统一性
- [D6] 角色弧光完整性（每个主要角色都有成长/变化）
- [D7] 支线与主线的耦合度
- [D8] 调性控制（tonal balance）

### 1.3 JOJO的奇妙冒险 — "黄金螺旋"创意法

**方法论：**
- 荒木飞吕彦的核心理念：角色的"意志力"决定一切
- 替身（Stand）设计原则：能力越受限，战斗越精彩（不是比谁更强，而是比谁更聪明）
- 每部换主角，但精神传承（乔斯达家族的"黄金精神"）
- 视觉化叙事：用画面/动作而非旁白传递信息
- 音乐性：命名系统（乐队名、歌曲名）创造文化质感

**揭示的评估维度：**
- [D9] 战斗/冲突设计的创意性（不是力量碾压，而是策略博弈）
- [D10] 能力系统的"约束美学"（限制创造精彩）
- [D11] 精神传承与系列一致性
- [D12] "展示而非讲述"的执行度

### 1.4 诡秘之主（网文）— "预设蓝图"法

**方法论：**
- 乌贼从一开始就设计了完整的22条非凡途径、22个序列体系
- 每条途径对应一个塔罗牌，有完整的升序/降序逻辑
- 多条叙事线并行（克莱恩的主线 + 各序列角色的支线）
- 伏笔极其密集：早期看似无关的细节在几十万字后回收
- 克制的爽点设计：不靠无脑升级，靠信息差和策略

**揭示的评估维度：**
- [D13] 世界观的系统性（不是零散设定，而是自洽体系）
- [D14] 多线叙事的管理能力
- [D15] 信息差驱动的叙事张力

### 1.5 道诡异仙（网文）— "叙事欺骗"法

**方法论：**
- 独创的"双重叙事"：读者以为的现实可能是幻觉
- 不可靠叙事者达到极致：主角自己都不知道什么是真的
- 恐怖氛围的精心构建：通过认知不确定性制造恐惧
- 东方民俗元素的深度融合（不是贴皮，是内化）

**揭示的评估维度：**
- [D16] 叙事层次的复杂度（单层/双层/多层叙事）
- [D17] 认知不确定性的管理
- [D18] 文化元素的内化程度

## 二、综合评估框架：12维度模型

基于以上方法论分析，提炼出12个核心评估维度。
分为4个层面：

### 层面A：结构工程（骨架）

| 维度 | 代号 | 定义 | 评估方法 |
|------|------|------|----------|
| 叙事弧线 | A1 | 故事是否有清晰的起承转合，是否有多个层次的弧线嵌套 | 自动（情感曲线分析）+ LLM Judge |
| 信息架构 | A2 | 信息释放的节奏：伏笔埋设→揭示→再埋设的循环是否精密 | LLM Judge + 伏笔回溯检测 |
| 节奏控制 | A3 | 张弛有度：高潮与缓冲的交替是否合理，是否避免了"灌水" | 自动（段落节奏分析）+ LLM Judge |
| 一致性 | A4 | 100万字内设定、人物、时间线不矛盾 | 自动（实体一致性检测）|

### 层面B：角色生态（血肉）

| 维度 | 代号 | 定义 | 评估方法 |
|------|------|------|----------|
| 角色弧光 | B1 | 主要角色是否有成长/变化/堕落，弧光是否完整 | LLM Judge（逐角色追踪）|
| 角色区分度 | B2 | 去掉名字能否分辨谁在说话，每个角色是否有独特的决策逻辑 | 自动（对话风格分析）+ LLM Judge |
| 关系网络 | B3 | 角色关系是否有动态变化，是否有复杂的情感/利益纠葛 | LLM Judge（关系图谱提取）|

### 层面C：叙事技艺（技巧）

| 维度 | 代号 | 定义 | 评估方法 |
|------|------|------|----------|
| 颠覆与反转 | C1 | 反转是否"意料之外，情理之中"，是否改变了读者对故事的理解 | LLM Judge |
| 展示而非讲述 | C2 | 世界观/情感/冲突是否通过行动和对话展示，而非旁白解释 | LLM Judge + 自动（叙述/对话比例）|
| 冲突设计 | C3 | 冲突是否多元（内在+人际+社会+宇宙），是否有策略性而非纯力量碾压 | LLM Judge |
| 文化质感 | D4 | 文化元素（民俗/哲学/历史）是否内化而非贴皮，是否增加了叙事厚度 | LLM Judge |

### 层面D：灵魂深度（意义）

| 维度 | 代号 | 定义 | 评估方法 |
|------|------|------|----------|
| 主题深度 | D1 | 是否有超越"善恶对立"的复杂主题，主题是否通过情节自然呈现而非说教 | LLM Judge |
| 情感共鸣 | D2 | 是否能触发真实的情感反应（不只是"爽"，还有恐惧、悲伤、敬畏、思考） | LLM Judge + 人类校准 |
| 原创性 | D3 | 在同类作品中是否提供了新的叙事方式、世界观或角色类型 | LLM Judge + n-gram对比 |

## 三、与创作方法论的对应关系

每个维度都可以追溯到具体的创作方法论：

```
创作方法论              →  评估维度
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
谜题盒叙事(进击的巨人)   →  A2 信息架构 + C1 颠覆与反转
等价交换结构(FMA)        →  A1 叙事弧线 + B1 角色弧光
黄金螺旋创意(JOJO)       →  C3 冲突设计 + C2 展示而非讲述
预设蓝图(诡秘之主)       →  A1 叙事弧线 + A4 一致性
叙事欺骗(道诡异仙)       →  A2 信息架构 + D1 主题深度
日更连载约束(网文本质)    →  A3 节奏控制 + A4 一致性
```

## 四、评估方法设计

### 4.1 三层评估 pipeline

```
文本输入（全书或指定章节集）
    ↓
Layer 1: 自动检测（零成本，秒级）
    ├── 情感弧线曲线 → A1 叙事弧线
    ├── 实体一致性检测 → A4 一致性
    ├── 叙述/对话/描写比例 → C2 展示而非讲述
    ├── 段落节奏分析 → A3 节奏控制
    ├── 对话风格聚类 → B2 角色区分度
    └── 伏笔关键词回溯 → A2 信息架构（辅助）
    ↓
Layer 2: LLM Judge（主要成本）
    ├── 维度 A1: 叙事弧线评估
    ├── 维度 A2: 信息架构评估
    ├── 维度 B1: 角色弧光评估（逐角色）
    ├── 维度 B3: 关系网络评估
    ├── 维度 C1: 颠覆与反转评估
    ├── 维度 C2: 展示而非讲述评估
    ├── 维度 C3: 冲突设计评估
    ├── 维度 C4: 文化质感评估
    ├── 维度 D1: 主题深度评估
    ├── 维度 D2: 情感共鸣评估
    └── 维度 D3: 原创性评估
    ↓
Layer 3: 聚合与校准
    ├── PCA 降维 → 综合分数
    ├── 各层面分数（A/B/C/D）
    ├── 与人类标注校准
    └── 百分位排名（在评测集中的位置）
```

### 4.2 LLM Judge Prompt 设计原则

**不要这样问：**
"这部小说好看吗？请打1-10分。" ← 太主观，不可复现

**要这样问（以 B1 角色弧光为例）：**

```
你是一位专业的叙事分析专家。请分析以下小说中角色 [角色名] 的弧光。

评估标准：
1. 该角色在故事开始时的核心特征是什么？（列出3-5个）
2. 该角色在故事结束时的核心特征是什么？（列出3-5个）
3. 这些特征中，哪些发生了变化？变化的原因是什么？
4. 变化是否由故事情节自然驱动，还是突兀/强行的？
5. 这个角色的成长/变化是否与故事主题相关？

请逐项回答，最后给出1-10分评分。
评分标准：
- 1-3分：扁平角色，无变化，纯工具人
- 4-5分：有变化但突兀/不合理
- 6-7分：有合理弧光，但深度一般
- 8-10分：弧光完整、深刻、与主题交织

注意：提供具体文本证据支持你的判断。
```

### 4.3 多评委系统

```
Judge 1: Claude (叙事分析 prompt)
Judge 2: GPT-4 (叙事分析 prompt)
Judge 3: Gemini (叙事分析 prompt)

最终分数 = 中位数

一致性检验：Kendall's W > 0.6 才采信
```

## 五、评测集设计

### 5.1 作品选择标准

选择 80-100 部作品，覆盖：

**S级（标杆作品，20部）：**
- 日本：进击的巨人、钢炼、JOJO、猎人、海贼王、死亡笔记、Code Geass、命运石之门
- 中文网文：诡秘之主、道诡异仙、凡人修仙传（前中期）、绍宋、赤色黎明、惊悚乐园
- 西方：冰与火之歌、黑暗物质三部曲、哈利波特、指环王、沙丘
- 韩国：神之塔、Sweet Home

**A级（优秀但有瑕疵，30部）：**
- 各类型中口碑较好但有明显问题的作品
- 如：斗破苍穹（后期崩坏）、完美世界（力量膨胀）、一拳超人（漫画版 vs 网文版）

**B级（普通/有明显缺陷，30部）：**
- 套路化严重、灌水明显、角色工具人的作品
- 如：大量同质化的系统文、签到文

### 5.2 采样策略

由于全书太长（网文可能500万字），采样策略：

```
采样 = 开头5章 + 中段5章 + 关键转折3章 + 结尾5章

开头：建立世界观、引入主角、设置第一个钩子
中段：故事发展、角色关系深化
转折：颠覆性事件（如进击的巨人墙外真相揭露）
结尾：收束、主题升华
```

## 六、独特之处：网文特有评估维度

与通用文学评估不同，网文需要额外关注：

### 6.1 连载稳定性

```
定义：在日更连载模式下，作品质量是否保持稳定

评估方法：
- 按时间序列分析各章节的各维度分数
- 计算分数方差：方差越大 = 质量波动越大
- 检测"烂尾指数"：后20%章节的分数是否显著下降

好的连载：诡秘之主（质量稳定）
烂的连载：斗破苍穹（后期严重下滑）
```

### 6.2 力量体系自洽性

```
定义：作品中的能力/力量系统是否逻辑自洽

评估方法（LLM Judge）：
1. 提取力量体系的核心规则（如：等价交换、斗气等级、序列途径）
2. 检测是否存在违反规则的情况
3. 评估力量体系是否有"天花板"（避免无限膨胀）
4. 检测"Deus Ex Machina"（突然出现新能力解决危机）

好的：FMA（等价交换始终有效）、JOJO（替身能力受限但创意十足）
烂的：大部分玄幻文（后期规则不断被打破）
```

### 6.3 读者粘性机制分析

```
定义：作品如何维持读者的持续阅读动力

评估维度：
- 钩子密度：每章结尾是否有有效的悬念/期待
- 信息差管理：读者 vs 角色的信息不对称是否被利用
- 爽点分布：是否有规律的满足感释放（不全是，也不能太久没有）
- 好奇心维护：是否持续提出新问题

这是网文特有的——传统文学不需要考虑这些。
但好的网文（诡秘之主）在这些维度上做得极好。
```

### 6.4 套路使用质量

```
定义：作品是"使用套路"还是"被套路绑架"

评估方法（LLM Judge）：
1. 识别作品使用了哪些类型套路（废柴逆袭、退婚流、系统流等）
2. 评估每个套路是否被创新性使用
3. 检测是否只是复制模板

好的使用：道诡异仙（用系统文框架讲了一个关于认知和存在的恐怖故事）
烂的使用：99%的系统文（完全模板化）
```

## 七、评分体系

### 7.1 维度权重

不是所有维度同等重要。权重由人类标注校准：

```
层面A: 结构工程      权重 30%
  A1 叙事弧线        8%
  A2 信息架构        8%
  A3 节奏控制        7%
  A4 一致性          7%

层面B: 角色生态      权重 25%
  B1 角色弧光       10%
  B2 角色区分度      8%
  B3 关系网络        7%

层面C: 叙事技艺      权重 25%
  C1 颠覆与反转      8%
  C2 展示而非讲述    6%
  C3 冲突设计        6%
  C4 文化质感        5%

层面D: 灵魂深度      权重 20%
  D1 主题深度        8%
  D2 情感共鸣        7%
  D3 原创性          5%
```

### 7.2 综合评级

```
S级 (9.0-10.0): 叙事杰作，教科书级别
A级 (7.5-8.9):  优秀作品，有明显亮点
B级 (6.0-7.4):  合格作品，无明显缺陷
C级 (4.0-5.9):  普通作品，有明显问题
D级 (1.0-3.9):  低质量，套路化/崩坏严重
```

## 八、Anti-patterns 检测清单

以下网文常见问题，benchmark 必须能检测到：

| 问题 | 检测维度 | 检测方法 |
|------|----------|----------|
| 灌水（无意义填充） | A3 节奏 | 段落信息密度分析 |
| 力量体系崩坏 | 力量体系自洽性 | LLM Judge + 规则提取 |
| 角色降智（为衬托主角） | B2 角色区分度 | 决策一致性分析 |
| 挖坑不填 | A2 信息架构 | 伏笔回收率检测 |
| 套路模板化 | D3 原创性 | n-gram对比 + LLM Judge |
| 烂尾 | 连载稳定性 | 后段分数趋势分析 |
| 人设崩塌 | B1 角色弧光 | 属性追踪 |
| 说教而非展示 | C2 展示而非讲述 | 叙述/对话比例 |

## 十、实施路线图

```
Phase 1 (2周): 数据集
  - 选 80 部作品
  - 提取采样章节
  - 建立自动检测 pipeline

Phase 2 (2周): LLM Judge
  - 设计 12 个维度的 judge prompt
  - 在 S 级作品上校准
  - 做评委一致性检验

Phase 3 (1周): 人类校准
  - 抽样 200 章节人工标注
  - 校准权重
  - 验证自动评分的区分度

Phase 4 (1周): 发布
  - 跑全量评测
  - 生成排行榜
  - 开源评测集和代码
```
